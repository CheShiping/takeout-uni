# 第三章：网络请求封装

## 3.1 Axios 封装

项目的 HTTP 请求统一封装在 `src/api/http.ts` 中，采用单例模式：

```typescript
import axios, { type AxiosInstance, type InternalAxiosRequestConfig, type AxiosResponse } from 'axios'
import { API_CONFIG } from '@/constants/api'
import { UniStorage } from '@/utils/StorageUtil'
import { ToastUtil } from '@/utils/ToastUtil'
import type { ApiResponse } from '@/models/api.types'

class HttpUtil {
  private instance: AxiosInstance

  constructor() {
    this.instance = axios.create({
      baseURL: API_CONFIG.BASE_URL,
      timeout: 15000,
      headers: { 'Content-Type': 'application/json' }
    })
  }
}

export const httpUtil = new HttpUtil()
```

设计要点：
- **单例模式**：全局只有一个 `httpUtil` 实例，避免重复创建
- **统一配置**：baseURL、超时时间、请求头集中管理
- **泛型支持**：所有请求方法都支持 TypeScript 泛型

## 3.2 Token 自动注入

通过请求拦截器，自动在每个请求头中附加 Token：

```typescript
this.instance.interceptors.request.use(
  (config: InternalAxiosRequestConfig) => {
    const token = UniStorage.getToken()
    if (token) {
      config.headers['satoken'] = 'Bearer ' + token
    }
    return config
  },
  (error) => {
    console.error('[HttpUtil] 请求拦截错误:', error)
    return Promise.reject(error)
  }
)
```

Token 使用 **Sa-Token** 认证框架的格式：`satoken: Bearer <token>`。

## 3.3 统一错误处理

响应拦截器处理所有返回数据：

```typescript
this.instance.interceptors.response.use(
  (response: AxiosResponse<ApiResponse>) => {
    const res = response.data
    if (res.code === 0) {
      return response           // 业务成功
    } else {
      ToastUtil.showError(res.message || '请求失败')
      return Promise.reject(new Error(res.message || '请求失败'))
    }
  },
  (error) => {
    // 网络错误、超时等
    const message = error?.response?.data?.message || error.message || '网络请求失败'
    ToastUtil.showError(message)
    return Promise.reject(error)
  }
)
```

约定的 API 响应格式：
```typescript
interface ApiResponse<T> {
  code: number      // 0 表示成功，非 0 表示失败
  message: string   // 提示信息
  data: T           // 业务数据
}
```

## 3.4 请求方法封装

提供 GET/POST/PUT/DELETE 四个方法，都支持泛型：

```typescript
async get<T>(url: string, params?: Record<string, unknown>): Promise<ApiResponse<T>> {
  const response = await this.instance.get<ApiResponse<T>>(url, { params })
  return response.data
}

async post<T>(url: string, body?: unknown, params?: Record<string, unknown>): Promise<ApiResponse<T>> {
  const response = await this.instance.post<ApiResponse<T>>(url, body, { params })
  return response.data
}
```

## 3.5 文件上传

uni-app 中文件上传不能使用 Axios，需要使用 `uni.uploadFile()`：

```typescript
async upload<T>(url: string, filePath: string, fileName?: string): Promise<ApiResponse<T>> {
  return new Promise((resolve, reject) => {
    const token = UniStorage.getToken()
    const header: Record<string, string> = {}
    if (token) {
      header['satoken'] = 'Bearer ' + token
    }

    uni.uploadFile({
      url: `${API_CONFIG.BASE_URL}${url}`,
      filePath: filePath,
      name: 'file',
      fileName: fileName || 'image.jpg',
      fileType: 'image',
      header,
      success: (uploadRes) => {
        const result: ApiResponse<T> = JSON.parse(uploadRes.data)
        if (result.code === 0) {
          resolve(result)
        } else {
          ToastUtil.showError(result.message || '上传失败')
          reject(new Error(result.message || '上传失败'))
        }
      },
      fail: (error) => {
        ToastUtil.showError('上传失败')
        reject(error)
      }
    })
  })
}
```

注意：`uni.uploadFile()` 返回的是字符串，需要手动 `JSON.parse` 解析。

## 3.6 API 常量管理

所有 API 路径集中在 `src/constants/api.ts` 中管理：

```typescript
export const API_CONFIG = {
  BASE_URL: import.meta.env.VITE_API_BASE_URL || '',

  PATHS: {
    // 食品
    FOOD_LIST: '/api/food/list',
    FOOD_DETAIL: '/api/food/{id}',
    FOOD_CATEGORIES: '/api/food/categories',

    // 购物车
    CART_LIST: '/api/cart/list',
    CART_ADD: '/api/cart/add',

    // 订单
    ORDER_CREATE: '/api/order/create',
    ORDER_LIST: '/api/order/list',

    // 用户
    USER_LOGIN: '/api/user/login',
    USER_REGISTER: '/api/user/register',
    // ... 更多路径
  }
} as const
```

使用 `as const` 确保类型推断为字面量类型，避免意外修改。

### 动态路径拼接

对于包含路径参数的 API，使用 `replace` 方法：

```typescript
// 使用示例
const url = API_CONFIG.PATHS.FOOD_DETAIL.replace('{id}', foodId)
const response = await httpUtil.get<FoodItem>(url)
```

## 3.7 Vite 代理配置

开发环境下，`API_CONFIG.BASE_URL` 为空字符串，请求会通过 Vite 代理转发：

```typescript
// vite.config.ts
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8123',
      changeOrigin: true
    }
  }
}
```

这样前端请求 `/api/food/list` 会被代理到 `http://localhost:8123/api/food/list`，避免跨域问题。

## 小结

本章讲解了网络请求层的设计。核心思想是**统一封装、集中管理**：Axios 单例处理所有 HTTP 逻辑，API 常量集中定义路径，拦截器自动处理 Token 和错误。下一章我们将讲解状态管理。
