# 第十七章：构建与发布

## 17.1 H5 生产构建

### 构建命令

```bash
npm run build:h5
```

构建产物在 `dist/build/h5` 目录，包含 `index.html`、CSS、JS 等静态文件。

### 部署方式

H5 产物是纯静态文件，可以部署到任何静态服务器：

**Nginx 配置示例：**

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/dist/build/h5;
    index index.html;

    # 解决 SPA 路由问题
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://localhost:8123;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|svg)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
```

### 环境变量

生产环境的 API 地址通过环境变量配置：

```bash
# .env.production
VITE_API_BASE_URL=https://api.your-domain.com
```

## 17.2 微信小程序构建与发布

### 构建命令

```bash
npm run build:mp-weixin
```

构建产物在 `dist/build/mp-weixin` 目录。

### 上传步骤

1. 打开**微信开发者工具**
2. 导入 `dist/build/mp-weixin` 目录
3. 在开发者工具中点击"上传"按钮
4. 填写版本号和项目备注
5. 登录 [微信公众平台](https://mp.weixin.qq.com/) 提交审核

### 小程序配置要点

在 `src/manifest.json` 中配置微信小程序的 AppID：

```json
{
  "mp-weixin": {
    "appid": "your-appid",
    "setting": {
      "urlCheck": false,
      "es6": true,
      "postcss": true
    },
    "usingComponents": true
  }
}
```

### 小程序域名白名单

在微信公众平台配置服务器域名：
- **request 合法域名**：你的后端 API 地址
- **uploadFile 合法域名**：文件上传地址
- **downloadFile 合法域名**：图片下载地址

## 17.3 App 打包与发布

### 云打包（推荐）

1. 在 HBuilderX 中打开项目
2. 菜单：发行 → 原生 App 云打包
3. 选择平台（Android / iOS）
4. 配置签名证书
5. 点击打包，等待完成

### 离线打包

需要安装 Android Studio（Android）或 Xcode（iOS），配置本地打包环境。

### App 配置

在 `src/manifest.json` 中配置 App 相关信息：

```json
{
  "app-plus": {
    "distribute": {
      "android": {
        "permissions": [
          "<uses-permission android:name=\"android.permission.ACCESS_FINE_LOCATION\"/>",
          "<uses-permission android:name=\"android.permission.ACCESS_COARSE_LOCATION\"/>"
        ]
      }
    },
    "modules": {
      "Maps": {},
      "Payment": {}
    }
  }
}
```

## 17.4 环境变量管理

### 多环境配置

```
.env                # 默认配置
.env.development    # 开发环境
.env.production     # 生产环境
```

```bash
# .env.development
VITE_API_BASE_URL=

# .env.production
VITE_API_BASE_URL=https://api.your-domain.com
```

在代码中使用：

```typescript
// src/constants/api.ts
export const API_CONFIG = {
  BASE_URL: import.meta.env.VITE_API_BASE_URL || '',
  // ...
}
```

- 开发环境：`BASE_URL` 为空，走 Vite 代理
- 生产环境：`BASE_URL` 为完整 URL，直接请求

## 17.5 性能优化建议

### 1. 图片优化

- 使用 CDN 托管图片
- 压缩图片体积
- 使用 `webp` 格式（小程序支持有限）
- 懒加载非首屏图片

### 2. 数据缓存

```typescript
// 缓存店铺列表，避免重复请求
const shopStore = useShopStore()
if (shopStore.shops.length === 0) {
  await shopStore.fetchShops()
}
```

### 3. 分页加载

```typescript
// 订单列表分页
async function loadMore() {
  page.value++
  await orderStore.fetchOrders(page.value, 10, currentStatus.value)
}
```

### 4. 组件懒加载

Tab 内容使用 `v-if` 而非 `v-show`，未激活的 Tab 不渲染：

```html
<HomeComponent v-if="currentTab === 0" />
<FoodCategory v-if="currentTab === 1" />
```

### 5. 减少 setData 频率

小程序中避免频繁更新响应式数据，合并更新：

```typescript
// 不好：多次更新
item.name = 'new name'
item.price = 10
item.count = 5

// 好：一次更新
Object.assign(item, { name: 'new name', price: 10, count: 5 })
```
