# 第十五章：跨平台适配

## 15.1 条件编译

uni-app 使用 `#ifdef` 和 `#ifndef` 指令进行条件编译，同一份代码可以根据平台编译出不同的产物。

### 语法格式

```typescript
// #ifdef MP-WEIXIN
// 只在微信小程序中编译
console.log('微信小程序环境')
// #endif

// #ifdef H5
// 只在 H5 中编译
console.log('H5 环境')
// #endif

// #ifdef APP-PLUS
// 只在 App 中编译
console.log('App 环境')
// #endif
```

### 平台标识

| 标识 | 平台 |
|------|------|
| `H5` | 网页 |
| `MP-WEIXIN` | 微信小程序 |
| `MP-ALIPAY` | 支付宝小程序 |
| `APP-PLUS` | App |
| `MP` | 所有小程序 |

## 15.2 权限管理（PermissionsUtil）

`src/utils/PermissionsUtil.ts` 是跨平台适配的典型案例，同一个方法在不同平台有不同的实现：

```typescript
export class PermissionsUtil {
  static async requestLocationPermission(): Promise<boolean> {
    return new Promise((resolve) => {
      // #ifdef MP-WEIXIN
      uni.getLocation({
        type: 'gcj02',
        success: () => resolve(true),
        fail: (err) => {
          if (err.errMsg?.includes('auth deny')) {
            uni.showModal({
              title: '权限提示',
              content: '需要获取您的位置信息来推荐附近店铺，请在设置中开启位置权限',
              success: (modalRes) => {
                if (modalRes.confirm) {
                  uni.openSetting({})  // 小程序打开设置页
                }
              }
            })
          }
          resolve(false)
        }
      })
      // #endif

      // #ifdef APP-PLUS
      uni.getLocation({
        type: 'wgs84',
        success: () => resolve(true),
        fail: () => {
          uni.showModal({
            title: '权限提示',
            content: '需要获取您的位置信息来推荐附近店铺'
          })
          resolve(false)
        }
      })
      // #endif

      // #ifdef H5
      if ('geolocation' in navigator) {
        navigator.geolocation.getCurrentPosition(
          () => resolve(true),
          () => {
            uni.showModal({
              title: '权限提示',
              content: '请在浏览器设置中允许访问您的位置信息'
            })
            resolve(false)
          }
        )
      } else {
        resolve(false)
      }
      // #endif
    })
  }
}
```

### 平台差异点

| 功能 | H5 | 微信小程序 | App |
|------|-----|-----------|-----|
| 位置坐标系 | wgs84 | gcj02 | wgs84 |
| 权限引导 | 浏览器设置 | `openSetting` | 系统设置 |
| API 来源 | `navigator.geolocation` | `uni.getLocation` | `uni.getLocation` |

## 15.3 文件上传的平台差异

```typescript
// src/api/http.ts 中的 upload 方法
async upload<T>(url: string, filePath: string): Promise<ApiResponse<T>> {
  return new Promise((resolve, reject) => {
    uni.uploadFile({
      url: `${API_CONFIG.BASE_URL}${url}`,
      filePath: filePath,
      name: 'file',
      header,
      success: (uploadRes) => {
        // uni.uploadFile 返回的是字符串，需要 JSON.parse
        const result: ApiResponse<T> = JSON.parse(uploadRes.data)
        // ...
      }
    })
  })
}
```

`uni.uploadFile` 在所有平台都可用，但返回值是字符串而非对象，这是需要注意的跨平台一致性问题。

## 15.4 存储 API 的平台适配

Pinia 持久化插件需要适配 uni-app 的存储 API：

```typescript
// src/main.ts
pinia.use(createPersistedState({
  storage: {
    getItem: (key: string) => uni.getStorageSync(key),    // 不是 localStorage
    setItem: (key: string, value: string) => uni.setStorageSync(key, value),
  }
}))
```

## 15.5 导航方式的统一

所有页面跳转统一使用 uni-app API，不依赖 vue-router：

```typescript
// 普通页面跳转
uni.navigateTo({ url: '/pages/food/food-detail?id=123' })

// 关闭当前页跳转
uni.redirectTo({ url: '/pages/mine/login' })

// 关闭所有页面跳转（用于 Tab 切换）
uni.reLaunch({ url: '/pages/main/main' })

// 返回上一页
uni.navigateBack()
```

## 15.6 样式适配

### rpx 单位

uni-app 中推荐使用 `rpx` 作为尺寸单位，自动适配不同屏幕：

```scss
.container {
  width: 750rpx;      // 满屏宽度
  padding: 20rpx;     // 自动缩放
  font-size: 28rpx;   // 自动缩放
}
```

### 安全区适配

```scss
.tab-bar {
  padding-bottom: constant(safe-area-inset-bottom);  /* iOS < 11.2 */
  padding-bottom: env(safe-area-inset-bottom);       /* iOS >= 11.2 */
}
```

## 小结

本章讲解了跨平台适配的核心技术。关键点是**条件编译 + 平台 API 差异处理 + 统一抽象**。通过 `#ifdef` 指令，同一份代码可以在不同平台上编译出正确的行为。下一章我们将讲解样式与主题。
