# 第二章：项目架构设计

## 2.1 整体架构分层

项目采用经典的分层架构，从上到下分为四层：

```
┌─────────────────────────────────┐
│         Pages（页面层）          │  用户看到的界面
├─────────────────────────────────┤
│      Components（组件层）        │  可复用的 UI 片段
├─────────────────────────────────┤
│   Hooks + Stores（逻辑层）       │  状态管理与业务逻辑
├─────────────────────────────────┤
│   Services + API（数据层）       │  服务调用与网络请求
└─────────────────────────────────┘
```

**数据流向**：`Pages → Hooks/Stores → Services → API → 后端`

每层职责清晰：
- **Pages**：只负责页面布局和用户交互，不直接调用 API
- **Components**：可复用的 UI 组件，通过 props 接收数据
- **Hooks**：组合式函数，封装可复用的业务逻辑
- **Stores**：Pinia 状态管理，维护全局或模块状态
- **Services**：静态服务类，封装后端 API 调用
- **API**：Axios 单例，处理 HTTP 请求/响应

## 2.2 路由系统

本项目有两套路由机制并存。

### uni-app 原生路由（主路由）

`src/pages.json` 是 uni-app 的核心路由配置，所有页面必须在这里注册：

```json
{
  "pages": [
    {
      "path": "pages/index/index",
      "style": { "navigationBarTitleText": "" }
    },
    {
      "path": "pages/main/main",
      "style": {
        "navigationBarTitleText": "外卖点餐",
        "navigationStyle": "custom"
      }
    },
    {
      "path": "pages/food/food-detail",
      "style": { "navigationBarTitleText": "菜品详情" }
    }
  ],
  "globalStyle": {
    "navigationBarTextStyle": "black",
    "navigationBarTitleText": "外卖点餐",
    "navigationBarBackgroundColor": "#F8F8F8"
  }
}
```

### vue-router（辅助路由）

`src/router/index.ts` 为 H5 SPA 模式提供 vue-router 支持，可配置路由守卫。但在实际页面跳转中，统一使用 uni-app 的 API：

```typescript
// 推荐的页面跳转方式
uni.navigateTo({ url: '/pages/food/food-detail?id=123' })  // 普通跳转
uni.redirectTo({ url: '/pages/mine/login' })                // 重定向
uni.reLaunch({ url: '/pages/main/main' })                   // 关闭所有页面
uni.switchTab({ url: '/pages/main/main' })                  // Tab 切换
```

## 2.3 全局配置

### manifest.json

应用的全局配置，包括 AppID、版本号、权限声明等。在 HBuilderX 中可视化编辑。

### vite.config.ts

```typescript
import { defineConfig } from 'vite'
import uni from '@dcloudio/vite-plugin-uni'
import path from 'path'

process.env.UNI_INPUT_DIR = path.resolve(__dirname, 'src')

export default defineConfig({
  plugins: [uni()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, 'src')
    }
  },
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:8123',
        changeOrigin: true
      },
      '/upload': {
        target: 'http://localhost:8123',
        changeOrigin: true
      }
    }
  }
})
```

关键配置：
- **`UNI_INPUT_DIR`**：指定 uni-app 源码目录为 `src`
- **`alias`**：`@/` 映射到 `src/`，方便引用
- **`proxy`**：开发时代理 `/api` 请求到后端服务器

## 2.4 路径别名

在 `tsconfig.json` 和 `vite.config.ts` 中都配置了路径别名：

```json
// tsconfig.json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

使用时：
```typescript
import { httpUtil } from '@/api/http'
import { useUserStore } from '@/stores/user.store'
import type { FoodItem } from '@/models/food.types'
```

## 2.5 TypeScript 配置

项目的 `tsconfig.json` 配置要点：
- `strict: true` 启用严格模式
- `target: ES2020` 编译目标
- `moduleResolution: bundler` 使用 Vite 的模块解析
- 包含 `src/env.d.ts` 提供 uni-app 全局类型声明

## 小结

本章介绍了项目的整体架构设计。核心思想是**分层解耦**：页面只关心布局，组件只关心 UI，Store 只关心状态，Service 只关心数据。下一章我们将深入网络请求层的封装。
