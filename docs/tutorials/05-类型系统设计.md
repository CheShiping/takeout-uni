# 第五章：类型系统设计

## 5.1 为什么用 TypeScript

在 uni-app 项目中使用 TypeScript 的核心价值：
- **编译期检查**：在运行前发现类型错误
- **智能提示**：IDE 自动补全属性和方法
- **重构安全**：修改接口时自动提示所有受影响的位置
- **文档作用**：类型定义本身就是最好的文档

## 5.2 按领域划分类型文件

项目将类型按业务领域拆分到 `src/models/` 目录：

```
models/
├── address.types.ts    # 地址相关
├── api.types.ts        # API 通用类型
├── cart.types.ts       # 购物车
├── common.types.ts     # 通用枚举
├── food.types.ts       # 食品
├── order.types.ts      # 订单
├── payment.types.ts    # 支付
├── review.types.ts     # 评价
├── search.types.ts     # 搜索
├── shop.types.ts       # 店铺
└── user.types.ts       # 用户
```

每个文件只包含该领域的类型，避免单文件过大。

## 5.3 API 通用类型

```typescript
// src/models/api.types.ts

/** 通用 API 响应 */
export interface ApiResponse<T = any> {
  code: number
  message: string
  data: T
}

/** 分页响应 */
export interface PageResponse<T> {
  list: T[]
  total: number
  page: number
  pageSize: number
}
```

`ApiResponse<T>` 使用泛型 `T`，每个接口调用时指定具体的 `data` 类型：

```typescript
// 使用示例
const response = await httpUtil.get<FoodItem[]>('/api/food/list')
// response.data 的类型自动推断为 FoodItem[]
```

### 继承与扩展

```typescript
/** 购物车食品项（继承 FoodItem） */
export interface CartFoodItem extends FoodItem {
  buyNum: number                          // 购买数量
  selectedSpecs?: Record<string, string>  // 选中的规格
  specId?: string                         // 规格 ID
  specName?: string                       // 规格名称
}
```

`CartFoodItem extends FoodItem` 意味着它拥有 `FoodItem` 的所有属性，并额外添加了购物车相关的字段。

## 5.5 订单类型与状态枚举

```typescript
// src/models/order.types.ts

/** 订单状态枚举 */
export enum OrderStatus {
  UNPAID = 'unpaid',
  PAID = 'paid',
  DELIVERING = 'delivering',
  COMPLETED = 'completed',
  CANCELLED = 'cancelled'
}

/** 状态中文映射 */
export const ORDER_STATUS_MAP: Record<OrderStatus, string> = {
  [OrderStatus.UNPAID]: '待支付',
  [OrderStatus.PAID]: '已支付',
  [OrderStatus.DELIVERING]: '配送中',
  [OrderStatus.COMPLETED]: '已完成',
  [OrderStatus.CANCELLED]: '已取消'
}
```

### 订单 Tab 类型

```typescript
export type OrderTabType = 'ALL' | 'PENDING' | 'COMPLETED' | 'CANCELED'

export const ORDER_TABS: OrderTab[] = [
  { key: 'ALL', label: '全部' },
  { key: 'PENDING', label: '待处理' },
  { key: 'COMPLETED', label: '已完成' },
  { key: 'CANCELED', label: '已取消' }
]
```

注意：`OrderTabType` 中的 `'CANCELED'`（单 L）是故意的，用于 Tab 分类；而 `OrderStatus.CANCELLED`（双 L）是枚举值，与后端保持一致。

### 订单筛选函数

```typescript
export function filterOrdersByTab(orders: Order[], tabType: OrderTabType): Order[] {
  switch (tabType) {
    case 'PENDING':
      return orders.filter(o =>
        o.status === OrderStatus.UNPAID ||
        o.status === OrderStatus.PAID ||
        o.status === OrderStatus.DELIVERING
      )
    case 'COMPLETED':
      return orders.filter(o => o.status === OrderStatus.COMPLETED)
    case 'CANCELED':
      return orders.filter(o => o.status === OrderStatus.CANCELLED)
    case 'ALL':
    default:
      return orders
  }
}
```

## 小结

本章讲解了项目的类型系统设计。核心原则是**按领域拆分、泛型复用、枚举约束**。好的类型系统能让代码更安全、更易维护。下一章我们将进入实际页面开发。
